# Clean State Checklist

Use this before ending a non-trivial session.

## Required Checks

- [ ] Startup path still works, or the breakage is recorded with risk.
- [ ] Verification path ran, or the skipped command and reason are recorded.
- [ ] `feature_list.json` reflects actual state. No item is `passing` without
  evidence.
- [ ] `current-state.md` records the current objective, trusted verification,
  touched files, blockers, and next step.
- [ ] No temporary debug files, stale generated reports, or undocumented partial
  work are left behind.

## Local Generated Output

Ignored generated directories are allowed when they are current build caches:

- `target/`: authoritative Rust workspace target directory.
- `ui/node_modules/`: npm dependency install cache.
- `src-tauri/gen/` and `src-tauri/permissions/autogenerated/`: Tauri-generated
  schema and permission authoring output. Keep ignored unless the Tauri
  authoring contract changes.

Clean these when they appear after verification:

- `src-tauri/target/`: stale nested Rust target directory. Tauri npm scripts
  set `CARGO_TARGET_DIR` to the absolute root `target/` path; new builds
  should use root `target/`.
- `ui/dist/`, `ui/playwright-report/`, and `ui/test-results/`: frontend build
  and test reports.

## Next Session

The next session should be able to read `AGENTS.md`, `feature_list.json`, and
`current-state.md`, then continue without reconstructing state from chat
history.
